# Challenge 06 — Policy Benefits Calculator

**Difficulty:** Intermediate · **Estimated time:** 2–4 hours  
**Status:** ✅ Completed

## Overview

A reusable TypeScript module that calculates insurance coverage for medical expenses against a policy definition. Expenses are processed **chronologically** so earlier claims consume annual limits, visit counts, and deductibles that affect later ones.

**Input:**
- [`src/data/policy.json`](./src/data/policy.json) — annual limits, per-visit sub-limits, copay, deductible, waiting periods, exclusions
- [`src/data/expenses.json`](./src/data/expenses.json) — 20 medical expenses exercising all rule types

**Output:**
- Per-expense decisions with human-readable reasons
- Summary with remaining balances per benefit type
- [`src/data/expected-results.json`](./src/data/expected-results.json) — expected output for all 20 expenses
- [`output/calculation-results.json`](./output/calculation-results.json) — regenerated by `npm run calculate`

## Usage (as a module)

```typescript
import { calculateBenefits } from './src/index.js';
import policy from './src/data/policy.json' with { type: 'json' };
import expenses from './src/data/expenses.json' with { type: 'json' };

const { results, summary } = calculateBenefits(policy, expenses);

for (const result of results) {
  console.log(result.expense_id, result.decision, result.reason);
}
```

## Rules Implemented

- Annual benefit limits (per benefit type)
- Per-visit sub-limits (`limit_per_visit`)
- Per-year sub-benefit limits (`limit_per_year`)
- Visit counts per sub-benefit (`visits_per_year`)
- Copay (percentage + `max_per_visit` cap)
- Annual deductible (applies to selected benefit types)
- Waiting periods (per benefit type)
- Diagnosis/sub-benefit exclusions
- Chronological processing with running state

## Test Coverage (13 tests)
